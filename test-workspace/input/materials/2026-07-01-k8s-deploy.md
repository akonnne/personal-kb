# Kubernetes 生产部署指南

## 核心概念

- **Pod** — 最小部署单元，包含一个或多个容器
- **Deployment** — 声明式更新 Pod 和 ReplicaSet
- **Service** — 稳定的网络端点，负载均衡
- **Ingress** — 七层路由，域名+路径转发

## 部署策略

| 策略 | 风险 | 回滚速度 | 适用场景 |
|------|------|----------|----------|
| Rolling Update | 低 | 慢 | 默认策略 |
| Blue-Green | 中 | 快 | 关键业务 |
| Canary | 低 | 快 | 灰度验证 |
| A/B Testing | 低 | 快 | 功能对比 |

## 配置管理

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  APP_ENV: production
  LOG_LEVEL: warn
```

## 监控告警

- **Prometheus** — 指标采集
- **Grafana** — 可视化面板
- **Alertmanager** — 告警路由
